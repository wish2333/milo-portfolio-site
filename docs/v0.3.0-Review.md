## 2. 架构与设计决策

### 2.1 个人主页使用独立路由

个人主页放在 `app/about/page.tsx`，不把简介、经验和工作流继续堆入已有项目画廊。这样可以保持首页“项目原型与产品判断”的单一任务，同时让 `/about` 成为适合快速浏览的个人信息页。

### 2.2 复用现有视觉系统，不复制项目画廊

个人主页复用米白纸张、深绿主色、暖金强调、衬线标题和噪点纹理，但采用经验列表、仓库卡片和工作流四步网格作为内容结构，避免把首页的项目原型交互重复一遍。

### 2.3 统计数据使用可追溯快照

概览区的两个统计不再使用与个人主页叙事重复的业务成果数字，改为 GitHub 公开贡献数据：2026.04—07 期间 442 次提交，涉及 10 个有更新的公开仓库。页面明确标注统计区间与快照日期，避免将静态数字误读为实时计数。

### 2.4 入口保持轻量

顶部入口使用与“代码主页”并列的文本按钮，不新增复杂菜单或客户端状态；桌面端位于代码主页左侧，移动端压缩为两个胶囊按钮，保持可发现性和可点击区域。

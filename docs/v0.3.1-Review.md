## 0. 结论

本次交付针对 v0.3.0 个人主页在桌面端的字号层级进行了可读性优化。标签、日期、来源、经历元信息和仓库元信息统一提升至 12px，正文说明统一为 14px；移动端保留原有字号和布局，首页项目展示区内部的模拟 UI 也不受影响。代码已通过 lint 与 diff hygiene 检查，可进入 PR 审阅与发布流程。

## 2. 架构与设计决策

### 2.1 使用桌面端媒体查询隔离调整

字号修正放在 `@media (min-width: 721px)` 中，并通过 `.about-page` 前缀限定范围。这样可以避免桌面端可读性优化意外改变移动端已经稳定的视觉比例，也不会影响首页的项目原型内容。

### 2.2 保留两级小字号层级

辅助标签、日期、数据来源和仓库元信息使用 12px；段落说明使用 14px。这样仍保留信息层级与密度，同时避免桌面显示器上 9–10px 文字阅读成本过高。

### 2.3 不放大模拟产品界面

首页项目卡片中的产品 UI 是作品展示的一部分，其 7–10px 文字用于模拟真实产品的信息密度。本次只调整个人主页，不改变这些展示内容的比例和细节感。
